## Getting Started

If you're new to YoFi.V3:

1. Start with the [Architecture Overview](ARCHITECTURE.md) to understand the system design
2. Review the [Contributing Guide](CONTRIBUTING.md) to set up your development environment
3. Read the [Architecture Decision Records](adr/README.md) to understand key design choices
4. Check the [TODO List](TODO.md) for areas that need work

## Contributing Documentation

When making significant architectural changes, please:

1. Document the decision in a new ADR using the [template](adr/template.md)
2. Update the [Architecture Overview](ARCHITECTURE.md) if the change affects system design
3. Update relevant README files in source directories
